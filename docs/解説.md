# YomiagePlayer 解説（Swift 初学者向け）

このドキュメントは、今回作成した読み上げアプリ **YomiagePlayer** のコードを
「Swift をはじめて触る人」でも読めるように、やさしく解説するものです。

順番に読めば、「このアプリがどう動いているか」「Swift / SwiftUI の基本」が
セットで分かるようになっています。

---

## 0. まず全体像

このアプリは、大きく分けて **3つの登場人物** で動いています。

```
┌─────────────────────────────┐
│  ContentView.swift          │  ← 画面（見た目・ボタン・スライダー）
│  「ユーザーが触る部分」       │
└──────────────┬──────────────┘
               │ ボタンを押す / スライダーを動かす
               ▼
┌─────────────────────────────┐
│  SpeechManager.swift         │  ← 頭脳（読み上げ・再生制御・状態管理）
│  「実際の処理をする部分」      │
└──────────────┬──────────────┘
               │ 文字を音声へ
               ▼
        iOS の読み上げ機能（AVSpeechSynthesizer）
```

それを補助する役割として、

- **YomiagePlayerApp.swift** … アプリの起動地点（一番最初に動く所）
- **DocumentTextExtractor.swift** … PDF / DOCX から文章を取り出す道具
- **ShareExtension/** … 共有シート（他アプリから文章を送る）用の小さな別プログラム

があります。

### ファイル一覧と役割

| ファイル | 役割 | たとえると |
|---|---|---|
| `YomiagePlayerApp.swift` | アプリの入口 | 建物の正面玄関 |
| `ContentView.swift` | 画面の見た目と操作 | お店の受付・カウンター |
| `SpeechManager.swift` | 読み上げの頭脳 | 厨房（実際に作る所） |
| `DocumentTextExtractor.swift` | ファイルから文字を抽出 | 翻訳係 |
| `ShareExtension/ShareViewController.swift` | 共有で受け取る | 出張窓口 |

> 💡 **設計のコツ**：「見た目（ContentView）」と「処理（SpeechManager）」を
> 分けています。こうすると、片方を直しても、もう片方が壊れにくくなります。
> これは **MVVM** という考え方に近いもので、iOS開発でよく使われます。

## 1. Swift の超基本（このコードに出てくるものだけ）

### 変数と定数

```swift
var x = 10     // var = 後で変えられる箱（変数）
let y = 20     // let = 一度入れたら変えられない箱（定数）
```

### 型（かた）

値には「種類」があります。

```swift
let name: String = "あいうえお"   // 文字列
let count: Int = 5               // 整数
let rate: Double = 1.5           // 小数
let on: Bool = true              // はい/いいえ（true / false）
```

`: String` の部分が **型注釈**。書かなくても Swift が推測してくれることも多いです。

### オプショナル（? が付くやつ）

「**値が無いかもしれない**」を表す Swift の特徴的な仕組みです。

```swift
var voiceID: String?     // String か、何も無い（nil）か
```

`?` が付いた値は、そのままでは使えず「中身があるか確認」してから使います。

```swift
if let id = voiceID {
    // ここでは id に必ず中身がある
}
```

このコードでも `selectedVoiceIdentifier: String?` のように多用しています。

### 関数（func）

処理のまとまりに名前を付けたもの。

```swift
func play() {
    // 再生する処理
}

func skip(by seconds: Double) {  // seconds という引数（入力）を受け取る
    // ...
}
```

### クラスと構造体（class / struct）

データと処理をまとめる「設計図」。

- `struct`（構造体）… 主に「値」を表す。SwiftUI の画面は struct で作ります。
- `class`（クラス）… 主に「ずっと存在し続けるもの」を表す。`SpeechManager` は class。

## 2. SwiftUI の基本（画面の作り方）

`ContentView.swift` は **SwiftUI** というフレームワークで書かれています。
ポイントは「**今の状態を書くと、画面が自動でその通りになる**」という考え方です。

### View（ビュー）＝ 画面の部品

```swift
struct ContentView: View {
    var body: some View {
        Text("こんにちは")   // 画面に「こんにちは」と表示
    }
}
```

`body` の中に書いたものがそのまま画面になります。

### よく使う部品

| 部品 | 何か |
|---|---|
| `Text("…")` | 文字を表示 |
| `Button { 処理 } label: { 見た目 }` | ボタン |
| `Slider(value: …)` | スライダー |
| `VStack { }` | 縦に並べる |
| `HStack { }` | 横に並べる |
| `ScrollView { }` | スクロールできるようにする |

今回の画面は、ざっくり

```
ScrollView {
  VStack {              ← 縦に積む
    TextEditor          （文章の入力欄）
    Button              （読み込み）
    HStack { Button Button }  （ペースト / ファイル）
    Slider              （シークバー）
    HStack { Button Button Button }（戻る・再生・進む）
    Slider              （速度）
    VStack { Picker }   （音声選択）
  }
}
```

という入れ子構造になっています（`ContentView.swift` の 22〜158行目）。

### @State ＝ 画面が持つ「状態」

```swift
@State private var sliderValue: Double = 0
```

`@State` を付けた変数が変わると、**SwiftUI が自動で画面を描き直します**。
たとえばシークバーの位置 `sliderValue` が変われば、つまみの位置も自動で動きます。

### データバインディング（$ 記号）

```swift
Slider(value: $sliderValue, in: 0...1)
```

`$sliderValue` の `$` は「**双方向のつながり**」を意味します。
ユーザーがスライダーを動かす → `sliderValue` が変わる、
逆にコードで `sliderValue` を変える → スライダーのつまみが動く、
という両方向の連動です。

## 3. 「画面」と「頭脳」のつながり（一番大事な所）

### ObservableObject と @Published

`SpeechManager` は「ずっと存在して、状態を持ち続ける頭脳」です。

```swift
final class SpeechManager: NSObject, ObservableObject {
    @Published private(set) var isPlaying = false
    @Published private(set) var progress: Double = 0
    ...
}
```

- `ObservableObject` … 「私の状態が変わったら画面に知らせます」という宣言
- `@Published` … この値が変わったら **画面に通知する** 印

つまり `SpeechManager` の `progress`（読み上げ進捗）が変わると、
それを見ている画面のシークバーが自動で動く、という仕組みです。

### @EnvironmentObject

```swift
struct ContentView: View {
    @EnvironmentObject private var speech: SpeechManager
```

画面側は「`speech` という頭脳を借りて使う」と書いています。
この `speech` を通じて `speech.play()` や `speech.progress` にアクセスします。

### 全体の通知の流れ（例：再生の進捗）

```
SpeechManager が progress を更新
        │（@Published が通知）
        ▼
ContentView が気づく
        │
        ▼
シークバーのつまみが自動で動く
```

このおかげで、「画面を手で描き直す」コードを書かなくて済みます。

## 4. YomiagePlayerApp.swift（入口）

```swift
@main
struct YomiagePlayerApp: App {
    @StateObject private var speech = SpeechManager()

    var body: some Scene {
        WindowGroup {
            ContentView()
                .environmentObject(speech)
        }
    }
}
```

- `@main` … 「ここがアプリの開始地点」という印
- `@StateObject private var speech = SpeechManager()` … 頭脳を **1つだけ** 作る
- `.environmentObject(speech)` … その頭脳を画面（ContentView）に渡す

> 💡 `@StateObject` は「このオブジェクトをここで生成して持ち続ける」、
> `@EnvironmentObject` は「上から渡されたものを借りる」。
> 生成は1か所（App）、利用は各画面、という役割分担です。

## 5. SpeechManager.swift（頭脳）の詳しい解説

ここがアプリの中心です。難しく見えますが、やっていることは
「**文章を文ごとに区切って、1文ずつ iOS に読ませる**」だけです。

### 5-1. なぜ「文ごと」に区切るのか？

iOS の読み上げ（`AVSpeechSynthesizer`）は、いったん喋り始めると
「途中の好きな位置から喋り直す」機能がありません。

そこで、文章を `。！？` などで **文に分割** し、1文ずつ読ませます。
こうすると「シークしたら、その文から読み直す」ことで、
擬似的に「好きな位置から再生」を実現できます。

```swift
static func splitIntoSentences(_ text: String) -> [String] {
    let delimiters: Set<Character> = ["。", "！", "？", "!", "?", "\n", "．", "."]
    var result: [String] = []
    var current = ""
    for ch in text {            // 1文字ずつ見ていく
        current.append(ch)      // 今の文に足す
        if delimiters.contains(ch) {   // 区切り文字なら
            result.append(current)     // 1文として確定
            current = ""               // 次の文へ
        }
    }
    if !current.isEmpty { result.append(current) }
    return result.isEmpty ? [text] : result
}
```

例：「こんにちは。元気ですか？」→ `["こんにちは。", "元気ですか？"]`

### 5-2. 位置の管理（どこまで読んだか）

「全体で何文字目まで読んだか」を数字で管理しています。

- `totalChars` … 文章全体の文字数
- `spokenGlobalOffset` … 今、全体の何文字目を読んでいるか
- `progress` … `spokenGlobalOffset ÷ totalChars`（0〜1の進捗）

シークバーの位置（0〜1）と、文章の文字位置を、この割り算で行き来します。

### 5-3. 再生・一時停止・シーク

```swift
func play() { ... }              // 再生（続きから or 先頭から）
func pause() { ... }             // 一時停止
func seek(toFraction:) { ... }   // 指定位置へ移動
func skip(by:) { ... }           // 15秒進む/戻る
```

`seek` の中身（要点）：

```swift
func seek(toFraction fraction: Double) {
    let target = ... // 0〜1 を「文字位置」に変換
    spokenGlobalOffset = target
    progress = fraction
    // 喋っている最中なら、いったん止めてから、その位置で読み直す
    ...
}
```

### 5-4. デリゲート（iOS からの「報告」を受け取る）

`AVSpeechSynthesizer` は、読み上げ中にこちらへ「報告」をくれます。
その報告を受け取る窓口が **デリゲート（delegate）** です。

```swift
extension SpeechManager: AVSpeechSynthesizerDelegate {

    // ① 読み上げ位置が進むたびに呼ばれる
    func speechSynthesizer(..., willSpeakRangeOfSpeechString ...) {
        // 進捗（progress）を更新 → 画面のシークバーが動く
    }

    // ② 1文を読み終えたら呼ばれる
    func speechSynthesizer(..., didFinish ...) {
        // 次の文へ進む
    }
}
```

> 💡 **シーク時の安定化**：`AVSpeechSynthesizer` は「停止（stop）してすぐ再生」を
> 繰り返すと内部状態が固まり、以後まったく喋らなくなることがあります（読み上げ中に
> シーク／早送りすると停止して固まる不具合の原因）。
> そこで **シーク・読み込み時はシンセサイザーを作り直す**（`resetSynthesizer()`）方式に
> しています。古いインスタンスはデリゲートを外して破棄するので、遅れて届く中断通知の
> 影響を受けず、毎回クリーンな状態から確実に読み直せます。

### 5-5. バックグラウンド再生とロック画面

```swift
private func configureAudioSession() {
    let session = AVAudioSession.sharedInstance()
    try? session.setCategory(.playback, mode: .spokenAudio, options: [])
}
```

- `.playback` … 画面を消しても・他アプリに移っても音を出し続ける設定
  （これ＋ `Info.plist` の `UIBackgroundModes: audio` でバックグラウンド再生が可能に）

ロック画面のコントロールは `MPRemoteCommandCenter` で設定します。

```swift
center.playCommand.addTarget { ... self?.play() ... }       // 再生ボタン
center.pauseCommand.addTarget { ... self?.pause() ... }     // 一時停止ボタン
center.changePlaybackPositionCommand.isEnabled = true       // ← シークバー
center.changePlaybackPositionCommand.addTarget { ... self.seek(...) ... }
```

`changePlaybackPositionCommand` を有効にすることで、
**ロック画面のシークバー（スクラブ）** が使えるようになります（これが当初の目的でした）。

そして、ロック画面に「今○分／全体△分」を表示するのが次の部分：

```swift
private func updateNowPlaying() {
    var info: [String: Any] = [:]
    info[MPMediaItemPropertyTitle] = "ページの読み上げ"
    info[MPMediaItemPropertyPlaybackDuration] = duration        // 全体の長さ
    info[MPNowPlayingInfoPropertyElapsedPlaybackTime] = elapsedTime  // 経過
    ...
}
```

> ⚠️ 読み上げには本当の「再生時間」が無いので、**文字数 × 想定秒数** で
> 擬似的な時間を作ってロック画面に見せています（`secondsPerChar` が変換係数）。

### 5-6. 速度の保存（UserDefaults）

```swift
@Published var speedMultiplier: Double {
    didSet {
        UserDefaults.standard.set(speedMultiplier, forKey: Self.speedKey)
    }
}
```

- `didSet` … 「この値が変わった**直後に**自動で実行される」仕掛け
- `UserDefaults` … アプリを閉じても残る、小さな保存箱（設定の保存に使う）

起動時には、保存した値を読み戻します（init の中）：

```swift
let saved = UserDefaults.standard.object(forKey: Self.speedKey) as? Double
speedMultiplier = saved.map { min(max($0, 1.0), 1.4) } ?? 1.0
```

「保存があればそれを（1.0〜1.4に丸めて）使い、無ければ 1.0」という意味です。

## 6. ContentView.swift（画面）の詳しい解説

### 6-1. シークバー

```swift
Slider(
    value: $sliderValue,
    in: 0...1,
    onEditingChanged: { editing in
        isEditingSlider = editing
        if !editing {                       // 指を離した瞬間に
            speech.seek(toFraction: sliderValue)  // その位置へシーク
        }
    }
)
```

ポイントは `isEditingSlider`。
ドラッグ中（指で操作中）は、再生側の進捗で勝手につまみが動かないように
「今ユーザーが触っているよ」という札を立てています。

```swift
.onReceive(speech.$progress) { p in
    if !isEditingSlider { sliderValue = p }   // 触っていない時だけ追従
}
```

### 6-2. 速度スライダー（倍率＋％表示）

```swift
Slider(value: $speech.speedMultiplier, in: 1.0...1.4, step: 0.05)
    .onChange(of: speech.speedMultiplier) { _ in
        speech.applySpeedChange()   // 再生中なら今の位置で読み直し
    }
```

ラベル：

```swift
private var speedLabel: String {
    let percent = Int((speech.speedMultiplier * 100).rounded())
    return String(format: "読み上げ速度  %.2f×（約%d%%）", speech.speedMultiplier, percent)
}
```

`1.0×（約100%）` のように表示します。

### 6-3. 音声選択（Picker）

```swift
Picker("音声", selection: $speech.selectedVoiceIdentifier) {
    ForEach(speech.availableVoices, id: \.identifier) { voice in
        Text(voiceLabel(voice)).tag(Optional(voice.identifier))
    }
}
```

- `ForEach` … 配列の要素ぶん、項目を繰り返し作る
- `tag(...)` … 各項目に「選ばれたら、この値」という識別を付ける

## 7. DocumentTextExtractor.swift（ファイルから文字を取り出す）

```swift
static func text(from url: URL) -> String? {
    switch url.pathExtension.lowercased() {
    case "pdf":  return textFromPDF(url)    // PDFKit で抽出
    case "docx": return textFromDOCX(url)   // zip を展開して抽出
    case "rtf":  ...
    default:     ...                        // テキストとして読む
    }
}
```

- **PDF** … iOS標準の **PDFKit** がページごとに文字を取り出してくれます（簡単・確実）。
- **DOCX** … iOS には DOCX を読む標準機能が無いので、
  「DOCX＝中身が zip」という性質を利用して、自分で zip を展開し、
  本文 XML（`word/document.xml`）から `<w:t>…</w:t>` のテキストを抜き出しています。
  （`switch` は「値によって処理を分ける」構文です）

> このファイルは「初学者がいきなり読むには難しめ」な部分（zip解凍など）を含みます。
> 最初は「PDF と DOCX から文字を取り出す係」とだけ理解すればOKです。

## 9. 処理の流れ（まとめ図）

### 再生ボタンを押したとき

```
[再生ボタン] (ContentView)
   → speech.togglePlayPause()
      → speech.play()
         → AVAudioSession を有効化
         → beginSpeaking() で最初の文を喋らせる
            → iOS が喋る
               → willSpeakRange で progress 更新 → シークバーが動く
               → didFinish で次の文へ
```

### シークバーを動かしたとき

```
[シークバーを離す] (ContentView)
   → speech.seek(toFraction:)
      → resetSynthesizer() でシンセサイザーを作り直す（確実に停止）
         → beginSpeaking() でシーク先の文から読み直す
```

## 11. 次の一歩（おすすめの学習・改造）

理解を深めるには、**小さく改造して動かす**のが一番です。例：

1. **音声も保存する**：速度（`speedMultiplier`）と同じやり方で、
   `selectedVoiceIdentifier` も `UserDefaults` に保存してみる。
2. **スキップ秒数を変える**：`skip(by: 15)` の `15` を `30` にしてみる。
3. **文の区切りを足す**：`splitIntoSentences` の区切り文字に `「`」`など追加してみる。
4. **ボタンの文言や色を変える**：`ContentView` の `Text` や `Label` をいじってみる。

> どれも1〜2行の変更で結果が変わるので、「変えて → Run → 確認」を
> 繰り返すと、コードと画面の対応が体で分かってきます。
